# 个性化日报总结 Prompt

对每篇候选文章，根据 `summary` 内容生成：
- **中文标题**（≤20字）：提炼核心信息点，不要照搬原标题直译。好标题 = 一句话说清楚「发生了什么 + 为什么值得关注」
- **描述**（2-4 行，总计 60-120 字）：用自然的段落感书写，语义连贯的内容放在同一行，不要机械地一句一行。包含：核心事实/观点、关键细节或数据、值得关注的原因。如有个性化关联（见下文）可自然融入

## 总结原则

- 用**信息密度高的中文**，不要翻译腔（避免"该文章讨论了…"、"作者认为…"）
- 推文类（`is_tweet`）内容简短，标题直接提炼观点，描述补充上下文即可
- 长文章提炼**最有价值的 1-2 个信息点**，不要面面俱到
- 如果原文是英文，标题和描述都用中文，专有名词保留英文（如 Claude、Figma）

## 个性化关联

总结时结合你对用户的记忆（MEMORY.md 中的偏好、近期对话、历史日报反馈），在描述中自然地加入个性化视角：

- 如果文章内容与用户近期讨论的话题相关，点明关联（如"和你上周聊的 XX 方案思路类似"）
- 如果文章涉及用户正在用的工具/技术栈，指出实用价值（如"你在用的 Obsidian 可以直接试"）
- 如果内容与用户之前表达的观点呼应或冲突，标注出来（如"和你之前说的 XX 观点相反，值得看看另一面"）
- **没有明确关联时不要硬凑**，跳过这一行即可
